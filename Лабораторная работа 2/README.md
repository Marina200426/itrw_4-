# Лабораторная работа 2: Структура классов товарной номенклатуры

## Описание

Реализована структура классов для ведения товарной номенклатуры с различными типами товаров и методами подсчета стоимости.

## Структура классов

### AbstractProduct (Абстрактный товар)
**Свойства:**
- id, name, basePrice, category, description, totalRevenue

**Методы:**
- Геттеры и сеттеры для свойств
- `calculateFinalPrice($quantity)` - абстрактный метод подсчета финальной стоимости
- `sell($quantity)` - метод продажи товара (вычисляет стоимость и добавляет к доходу)
- `getTotalRevenue()` - получение общего дохода с продаж
- `getInfo()` - получение информации о товаре

### DigitalProduct (Цифровой товар)
**Наследник:** AbstractProduct

**Особенности:**
- Стоимость всегда в **2 раза меньше** базовой цены реального товара
- Метод `calculateFinalPrice($quantity)` возвращает: `(basePrice / 2) * quantity`
- Дополнительный метод: `getDigitalPrice()` - получение цены цифрового товара

**Пример:**
- Базовая цена: 1000 руб.
- Цена цифрового товара: 500 руб.
- За 2 шт.: 1000 руб.

### PhysicalProduct (Штучный физический товар)
**Наследник:** AbstractProduct

**Особенности:**
- Стоимость зависит от количества штук
- Метод `calculateFinalPrice($quantity)` возвращает: `basePrice * quantity`
- Дополнительный метод: `getPricePerUnit()` - получение цены за штуку

**Пример:**
- Цена за штуку: 1000 руб.
- За 2 шт.: 2000 руб.
- За 5 шт.: 5000 руб.

### WeightProduct (Товар на вес)
**Наследник:** AbstractProduct

**Особенности:**
- Стоимость зависит от продаваемого количества в килограммах
- Метод `calculateFinalPrice($quantity)` возвращает: `basePrice * quantity` (где quantity - кг)
- Дополнительный метод: `getPricePerKilogram()` - получение цены за 1 кг

**Пример:**
- Цена за 1 кг: 150 руб.
- За 2.5 кг: 375 руб.
- За 0.5 кг: 75 руб.

## Формирование дохода с продаж

Все классы имеют свойство `totalRevenue`, которое автоматически увеличивается при каждом вызове метода `sell($quantity)`. Метод `sell()`:
1. Вычисляет финальную стоимость через `calculateFinalPrice($quantity)`
2. Добавляет эту сумму к общему доходу
3. Возвращает финальную стоимость

- `PhysicalProduct.php` - класс штучного физического товара
- `WeightProduct.php` - класс товара на вес
- `index.php` - примеры использования и сравнения всех типов товаров

## Особенности реализации

1. **Абстрактный класс** - определяет общую структуру и требует реализации метода `calculateFinalPrice()` в наследниках
2. **Полиморфизм** - каждый тип товара по-своему вычисляет финальную стоимость
3. **Инкапсуляция** - доход с продаж управляется через защищенный метод `addRevenue()`
4. **Единый интерфейс** - все товары имеют одинаковый метод `sell()` для продажи
